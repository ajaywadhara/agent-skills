# Component Guide

Every external service and widget available for profile READMEs, with integration URLs and reliability notes.

---

## Tier 1: Reliable (99%+ uptime, use freely)

### Shields.io Badges

**Service:** `img.shields.io`
**Reliability:** Excellent — backed by CDN, rarely fails.

```markdown
<!-- Static badge -->
![Label](https://img.shields.io/badge/LABEL-VALUE-COLOR?style=flat-square&logo=LOGO&logoColor=white&labelColor=BG)

<!-- Dynamic GitHub badge -->
![Stars](https://img.shields.io/github/stars/USER/REPO?style=flat-square&color=ACCENT&labelColor=BG)
![Forks](https://img.shields.io/github/forks/USER/REPO?style=flat-square&color=MUTED&labelColor=BG)
![Followers](https://img.shields.io/github/followers/USER?style=flat-square&logo=github&logoColor=white&labelColor=BG&color=ACCENT)
```

**Style options:** `flat`, `flat-square`, `for-the-badge`, `plastic`, `social`
**Use `flat-square` as default** — cleanest, most professional.

### Skill Icons

**Service:** `skillicons.dev`
**Reliability:** Excellent — static SVG icons, fast CDN.

```markdown
<!-- Row of icons (max 15 per row) -->
<a href="https://skillicons.dev">
  <img src="https://skillicons.dev/icons?i=java,spring,ts,react,aws,docker&theme=dark" />
</a>
```

**Parameters:**
- `i=` — comma-separated icon IDs
- `theme=dark` or `theme=light`
- `perline=` — icons per row (default: all in one line)

**Common icon IDs:**
- Languages: `java`, `python`, `ts`, `js`, `go`, `rust`, `cpp`, `cs`, `ruby`, `php`, `swift`, `kotlin`
- Frontend: `react`, `nextjs`, `vue`, `svelte`, `angular`, `tailwind`, `html`, `css`
- Backend: `spring`, `django`, `flask`, `express`, `nestjs`, `fastapi`, `rails`
- Cloud: `aws`, `gcp`, `azure`, `cloudflare`, `vercel`, `netlify`, `heroku`
- Data: `postgres`, `mysql`, `mongodb`, `redis`, `dynamodb`, `elasticsearch`, `kafka`
- DevOps: `docker`, `kubernetes`, `terraform`, `jenkins`, `githubactions`, `nginx`
- Tools: `git`, `github`, `vscode`, `idea`, `vim`, `neovim`, `linux`, `bash`
- AI: `pytorch`, `tensorflow`

**Auto-detection:** Map repo `primaryLanguage` values to skill icon IDs:
- Java → `java`, `spring` (if Spring Boot detected in description)
- TypeScript → `ts`
- JavaScript → `js`
- Python → `python`
- Go → `go`
- Rust → `rust`
- Shell → `bash`

### Profile View Counter

**Service:** `komarev.com/ghpvc`
**Reliability:** Excellent.

```markdown
<img src="https://komarev.com/ghpvc/?username=USER&style=flat-square&color=ACCENT&labelColor=BG" />
```

### Readme Typing SVG

**Service:** `readme-typing-svg.demolab.com`
**Reliability:** Good — hosted on Vercel, occasionally slow.

```markdown
<img src="https://readme-typing-svg.demolab.com?font=JetBrains+Mono&weight=500&size=20&duration=3000&pause=1500&color=ACCENT&center=true&vCenter=true&repeat=true&width=600&height=35&lines=LINE1;LINE2;LINE3" />
```

**Parameters:**
- `font` — `JetBrains+Mono` (recommended for terminal feel) or `Inter` (clean sans)
- `weight` — `500` or `600`
- `size` — `18`-`24`
- `duration` — typing speed in ms per line (`3000` default)
- `pause` — pause between lines (`1500` default)
- `color` — hex without `#`
- `center=true&vCenter=true` — center alignment
- `width` — container width
- `lines` — semicolon-separated, URL-encoded

**Style as terminal commands for dev profiles:**
```
lines=$+building+distributed+systems+→;$+shipping+ai+agents+→;$+open-sourcing+dev+tools+→
```

## Tier 2: Mostly Reliable (occasional rate limits)

### GitHub Readme Streak Stats

**Service:** `github-readme-streak-stats.herokuapp.com`
**Reliability:** Good but can be slow during peak hours.

```markdown
<img src="https://github-readme-streak-stats.herokuapp.com/?user=USER&theme=github-dark-blue&background=BG&border=BORDER&stroke=BORDER&ring=ACCENT&fire=ACCENT_WARM&currStreakLabel=TEXT&sideLabels=MUTED&dates=BORDER" height="200" />
```

### GitHub Readme Activity Graph

**Service:** `github-readme-activity-graph.vercel.app`
**Reliability:** Good. Use `<picture>` tag for dark/light variants.

```markdown
<picture>
  <source media="(prefers-color-scheme: dark)" srcset="https://github-readme-activity-graph.vercel.app/graph?username=USER&bg_color=BG&color=ACCENT&line=ACCENT&point=TEXT&area=true&area_color=ACCENT&hide_border=true" />
  <source media="(prefers-color-scheme: light)" srcset="https://github-readme-activity-graph.vercel.app/graph?username=USER&bg_color=ffffff&color=ACCENT&line=ACCENT&point=1f2328&area=true&area_color=ACCENT&hide_border=true" />
  <img src="https://github-readme-activity-graph.vercel.app/graph?username=USER&bg_color=BG&color=ACCENT&line=ACCENT&point=TEXT&area=true&area_color=ACCENT&hide_border=true" width="100%" />
</picture>
```

### Snake Animation (Self-hosted via GitHub Action)

**Service:** Generated by `Platane/snk` GitHub Action → stored on `output` branch.
**Reliability:** Excellent once set up (self-hosted SVG).

```markdown
<picture>
  <source media="(prefers-color-scheme: dark)" srcset="https://raw.githubusercontent.com/USER/USER/output/github-snake-dark.svg" />
  <source media="(prefers-color-scheme: light)" srcset="https://raw.githubusercontent.com/USER/USER/output/github-snake.svg" />
  <img src="https://raw.githubusercontent.com/USER/USER/output/github-snake-dark.svg" alt="Snake animation" width="100%" />
</picture>
```

## Tier 3: Unreliable (DO NOT USE)

### GitHub Readme Stats — Pin Cards

**Service:** `github-readme-stats.vercel.app/api/pin`
**Status:** AVOID. Frequently rate-limited on the public Vercel instance. Images show as broken.

### GitHub Profile Trophy

**Service:** `github-profile-trophy.vercel.app`
**Status:** AVOID. Same rate-limit issues as above.

### GitHub Readme Stats — Stats Card

**Service:** `github-readme-stats.vercel.app/api`
**Status:** AVOID for the same reason. Use streak stats + activity graph instead for stats.

**Alternative for stats:** Use shields.io dynamic badges which are far more reliable:
```markdown
![Commits](https://img.shields.io/github/commit-activity/y/USER/REPO?style=flat-square&label=commits/year)
```

## Custom Header SVG

The header SVG is a custom file committed to the repo. It does NOT use any external service.

**Template structure:**
```xml
<svg width="1200" height="300" xmlns="http://www.w3.org/2000/svg">
  <defs>
    <!-- Background gradient using theme bg + bg-secondary -->
    <!-- Accent glow using theme accent (low opacity) -->
    <!-- Grid pattern using theme border color -->
    <!-- Text glow filter using theme accent -->
  </defs>

  <!-- Background rect with gradient -->
  <!-- Grid overlay (opacity 0.3-0.5) -->
  <!-- Glow orbs (2 max, using accent + accent-blue) -->
  <!-- Floating particles (6-8, using accent color, with animate) -->

  <!-- Name text: theme text color, 50-56px, sans-serif, centered -->
  <!-- Role text: theme accent color, 16-20px, monospace, UPPERCASE, letter-spacing 4 -->
  <!-- Tagline text: theme text-muted, 14-16px, sans-serif, centered -->

  <!-- Bottom accent line with gradient fade -->
  <!-- Corner bracket accents (4 corners, theme accent, opacity 0.3) -->
</svg>
```

**Note on animations:** GitHub sanitizes SMIL animations from repo SVGs. Include `<animate>` elements anyway — they work on other platforms (npm, GitLab) and the SVG looks complete in raw view. Design so it looks great as a static image too.
