# Theme Catalog

Each theme defines a consistent color palette used across every component in the profile.

## Theme: `github-dark`

GitHub's native dark palette. Clean, familiar, professional.

| Role | Hex | Usage |
|------|-----|-------|
| `bg` | `#0d1117` | SVG background, card backgrounds, badge labelColor |
| `bg-secondary` | `#161b22` | Subtle surfaces, gradient midpoints |
| `border` | `#30363d` | Card borders, divider lines |
| `text` | `#e6edf3` | Primary text, headings |
| `text-muted` | `#8b949e` | Secondary text, descriptions |
| `accent` | `#238636` | Links, highlights, primary accent |
| `accent-warm` | `#FF6B35` | Fire/streak icon, warm callouts |
| `accent-blue` | `#1f6feb` | Secondary accent, subtle glow |

**Typing SVG color:** `238636`
**Streak ring/fire:** `ring=238636`, `fire=FF6B35`
**Activity graph:** `color=238636`, `line=238636`, `point=e6edf3`, `area_color=238636`

## Applying a Theme

Every color reference in the generated README must use the selected theme's palette. This includes:

1. **Header SVG** — `bg`, `bg-secondary` for background gradient, `text` for name, `accent` for role subtitle, `text-muted` for tagline
2. **Typing SVG** — `color` parameter = `accent` (without #)
3. **Shields.io badges** — `labelColor` = `bg` (without #), `color` = `accent` (without #)
4. **Streak stats** — `background` = `bg`, `border` = `border`, `ring` = `accent`, `fire` = `accent-warm`, `currStreakLabel` = `text`, `sideLabels` = `text-muted`, `dates` = `border`
5. **Activity graph** — `bg_color` = `bg`, `color` = `accent`, `line` = `accent`, `point` = `text`, `area_color` = `accent`
6. **Komarev counter** — `color` = `accent`, `labelColor` = `bg`
